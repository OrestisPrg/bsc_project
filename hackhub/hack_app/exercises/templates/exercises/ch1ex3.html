What is Ethical Hacking?

Ethical hacking or Penetration Testing can be defined as the act of breaking into a system with the owner's permission. The purpose of intruding a system is to detect, identify, report and possibly fix vulnerabilities and weaknesses. Ethical hackers are trusted individuals that adopt the techniques procedures, tools and knowledge used by malicious hackers to identify a system's vulnerabilities and prevent their exploitation.

Phases of Ethical Hacking

The process of penetration testing can be usually broken down to 5 steps:

  1. Reconnaissance / Footprinting
  2. Scanning
  3. Gaining Access
  4. Maintaining Access
  5. Clearing Tracks

Reconnaissance / Footprinting

In this initial stage, the hacker focuses on gathering as much information as possible about the target system before launching an attack. During the process of footprinting, the hacker collects valuable information about the target system by actively interacting with it, usually through a network. The data collected usually include IP addresses, system names and the domain name, TCP and UDP services and passwords. With this information, the attacker is able to identify potential system vulnerabilities and how they can be exploited.

Scanning

In this phase, the attacker identifies the system's vulnerabilities and weaknesses which can be exploited, by gaining access to the network and collecting information. The information gathered during footprinting can be utilised to aid this procedure. There are typically 3 types of scanning involved in this phase each of which focuses on a specific set of vulnerabilities.

  • Port scanning: the target is scanned with port scanners, vulnerability scanners and other data-gathering equipment in order to obtain information like open ports, live systems, and services running on the host.
  • Vulnerability Scanning: the target is checked for exploitable vulnerabilities and weaknesses with help of automated tools.
  • Network Mapping: collecting information about the host, topology of network, routers, servers, and firewalls.

Gaining Access

In this phase, the hacker gains access to the system, applications, and network with the aid of certain tools and procedures. Upon entering the target, the hacker needs to elevate their privileges to a system administrator, in order to be able to fully control the system, manipulating data and installing software.

Maintaining Access

If the hacker's purpose goes beyond accessing the system and particularly wants to mantain a persistent connection between the comporomised system and their system, they need to secure access to the system using specific software in order to carry out an intended task.

Clearing Tracks

In the final phase, the hacker clears all evidence in order to remove traces that can lead to their identity. The process involves tampering or deleting log files and their content, clearing the cache and cookies, closing all the open ports and unintalling / deleting any software and files used.
