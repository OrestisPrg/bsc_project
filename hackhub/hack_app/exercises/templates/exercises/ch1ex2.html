Users

A UNIX-like system's user can either be a regular user, an administrative user or a service account.

  • Root/Superuser: An administrative root account, commonly known as a superuser, is automatically created and has administrative privileges for all services, access to all commands and files on the system.
  • Regular Users: have the necessary privileges to perform standard tasks on a Linux computer such as running word processors, databases, and Web browsers. They can store files in their own home directories. Since regular users do not normally have administrative privileges, they cannot accidentally delete critical operating system configuration files.
  • Service Accounts: are usually created and configured when service software is installed by the packet manager. Through their individual accounts, services can interact with the operating system. Common examples of service accounts include email, printing and web-sever software's accounts.

Sudo

While a superuser is authorised to use all commands and has access to all sensitive files on the system, regular users have limited access to particular files and directories (such as system files) and to which commands they can use and where. Limiting the power of regular users, can add a layer of security, protecting the system and its files from both malicious exploitation of the administrative priviledges and accidental misusage of those priviledges.

The sudo command can be used by a regular user to run a command with the priviledges of a sustem administrator/superuser. For example, in cases where a user needs to access, modify or even remove files they don't have the permission to, sudo can be used to elevate the user's privilidges and allow them to perform the desired action.

Usage: Before typing a command that requires root privileges to be carried out, the user must initially type sudo in a the terminal: sudo [command]

You must always perform tasks working as a regular user unless the directory or files you are working on require root priviledges. Working as root on regular directories/files will not only change their permissions to root, but prevents you from accidentally deleting or altering critical operating system files.

Permissions

While practising ls in the previous section, you might have noticed that ls -l outputs a string of characters for each of the working directory's contents as can be seen in the screenshot below.



This string represents the file's type and permissions and can be broken down to 4 components:

  1. The first character specifies if the subject is:
    • d a directory
    • - a file
    • l a symbolic link, which is file containing a reference to another file/directory as a path. It can be thought of as a shortcut in Windows.
  2. Characters 2-4 represent the owner's permissions:
    • r indicates the owner can read the file or - otherwise.
    • w indicates the owner can modify the file or - otherwise.
    • r indicates the owner can execute the file or - otherwise.
  3. Characters 5-7 represent the permissions of the group's members that own the file, denoted with rwx , exactly as the owner's permissions.
  4. Characters 8-10 represent the permissions for the rest of the users, denoted with rwx as the other user categories.

Changing Permissions & Owner

You can change the permissions of a file or multiple files for your user account using the chmod command. The permission modifications are defined by using the + or - symbol denoting the addition or removal of a permission corre spondingly, immediately followed by the permission type ( r, w, x) You can also change the group or global permissions of the file by using the letters g or a right before the +/- symbols.

Usage: chmod [a/g][+/-][r/w/x] [filename(s)]
Example: chmod g+w passwords.txt will enable the group of owners to modify the passwords.txt file.

The owner of a file or multiple files can also be change with the chown command.

Usage: chown [new_owner] [filename(s)]
