Essential Tools

2.1: Setting up your Hacking Environment

This chapter will introduce you to some of the most popular and essential command-line tools for any penetration tester. Certain tools, such as Nmap, can be considered a crucial part of footprinting and therefore, are preinstalled in OS distributions dedicated to penetration testing (such as Kali Linux).

In order to familiarise yourself with the tools as well as experiment and observe their actual functionality, you will need to target another system. Of course, performing such tasks on other real systems not only is illegal unless explicit permission from the owner has been provided, but can cause permanent damage to the system accidentally or even intentionally in some cases. It is therefore crucial to work with disposable, simulated systems, that have been created for this purpose such as virtual machines.

Working with Virtual Machines

The benefits of working with a virtual machine may already be clear from using the Kali Linux VM, especially when it comes to ethical hacking. As mentioned above, virtual machines provide a sandbox environment to the user encouraging experimentation. Therefore, users can practise their hacking skills by targeting a VM especially designed to be attacked and hacked. Ideally, the VM will intentionally be vulnerable allowing users to detect and exploit a number of different vulnerabilities.

Metasploitable 2

Metasploitable 2 is an intentionally vulnerable virtual machine with a version of Ubuntu Linux OS, designed to provide a vulnerable and easily exploitable target for ethical hackers. It was created by Rapid7, the owners of the Metasploit Framework which is one of the leading penetration testing frameworks and it's mainly intended to be used in combination with Metasploit. Chapter 3 will focus on the Metasploit Framework and demonstrate how it can be used to successfully penetrate metasploitable. For the moment, we will use it as a target to put the skills gained from each section into practice.

Installation and Configuration:

  1. Download the Metasploitable 2 image. You can do this using this link.
  2. Once the download is complete, navigate to VirtualBox and click on the New button at the top left.
  3. You can name this "metasploitable" and you should set the type to "Ubuntu" and version to "Ubuntu(64-bit)".
  4. Select the desired amount of RAM to be allocated. 2048 MB should be enough.
  5. In the "Hard Disk" screen, select the "Use an existing virtual disk image" option and browse your files to find the image downloaded earlier.
  6. By selecting create, your virtual machine should be created.
Once your have started the Metasploitable VM, you will be prompted to login. Use "msfadmin" for both username and password.

With metasploitable created, you will now need to run both Kali Linux and this in parallel in order to be able to experiment on it. In order to achieve this each VM must obtain its own IP address. This is achieved following the steps below:
  1. In VirtualBox main screen, click on the "Global Tools" button located on the top right.
  2. Then, select "Host Network Manager" from the available options.
  3. On the next screen, click on the "Create" button on the top left.
  4. A new item should appear below signifying that a host-only network has successfully been created.
  5. Make sure that DHCP Server is enabled, listed as an attribute of the item created.
  6. Go back to the main screen by pressing the "Machine Tools" button.

    7. Do the following for both VMs:

  7. Select the VM and click on the "Settings" button on the top left.
  8. Select the "Network" tab from the menu on the left and then the "Adapter 2" tab from the on-screen tab list.
  9. Enable the Network Adapter and set attached to to "Host-only Adapter" from the dropdown menu.
  10. Set "name" to the network adapter created in step 4 and click OK.
After following the above, you should have 2 virtual machines with different IP addresses, connected to a virtual network. You can check this by using the ifconfig command. Each machine's IP address can be found in the "eth0" block labeled as "inet".

Warning: After powering on your Kali VM, you may need to manually enable the network adapter that connects you to the internet. You can do that by clicking "Connect" on the corresponding network adapter, on the upper taskbar network settings.
