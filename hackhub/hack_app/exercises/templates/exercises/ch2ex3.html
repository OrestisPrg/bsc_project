2.3: Packet Analysers: Wireshark & tcpdump

Take Quiz

Packet analysers are software specialised in monitoring and intercept network traffic. Packets that pass over the network can be logged and itercepted, in a process called packet capture. A packet analyser can decode the captured package's raw data, revealing the information contained in the package often relevant to a user's activity. The contained information include addresses, visited websites, protocols used, dates, headers etc. Captured packages and traffic are usually logged in .pcap files.

tcpdump

tcpdump is a command-line packet analyser which has the ability to log TCP/IP, UDP and a lot more packages over the network the machine is connected to. Any unencrypted traffic passing through a system can be intercepted by a user with elevated priviledges, providing valuable information such as user credentials. Tcpdump is pre-installed on Kali.

Usage

You can start viewing and capturing packets by simply using the tcpdump command. By default, it will monitor traffic from the "eth0" interface by you can specify another interface using the -i option.

  • Save traffic in a .pcap file with: tcpdump -w [filename]
  • Read captured traffic from a file: tcpdump -r [filename]
  • Filter captured traffic for packets to/from a specific host: tcpdump -r [filename] host [ip_address]
  • Prevent packet convertion and DNS resolution: tcpdump -nn

Wireshark

Wireshark is also a packet analyser with a very similar functionality to tcpdump with the difference that it provides a GUI. Due to its graphical interface, Wireshark provides filtering and sorting that makes the task of finding a specific packet much easier compared to the command-line based tcpdump. Often capturing traffic with tcpdump might be an easier approach, which can then be analysed using Wireshark's interface. You can find Wireshark on your Kali VM.
