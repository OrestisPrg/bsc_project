Getting Started

Ethical Nature

Both malicious and ethical hackers share common procedures, techniques and tools that allow them to successfully break into a remote system. The purpose of a malicious hacker is to compromise a system in order to steal, destroy and modify data, alter the system's functionality or perform any other task that will disrupt the system's normal beahaviour. In contrast, an ethical hacker's purpose is to identify vulnerabilities in a target system without damaging the system itself and fix them, or report them to their employer. Therefore, an ethical hacker must always have the explicit concent of the target system's owner in order for their actions to be legal. Attempting or breaking into a system without the owner's full permission is illegal even if your intentions are to help them.

Please be aware that HackHub's purpose is to provide knowledge and skills that must be used ONLY in an ethical manner. Again, using these skills otherwise is illegal and criminally prosecuted.

About HackHub

Hackhub is an online educational platform that aims at introducing people from all backgrounds to ethical hacking.

  • The website's resources are divided into chapters, which categorise different topics based on their content and their difficulty in terms of required prior knowledge.
  • Each chapter is divided into sections focusing on a specific topic and skill. Studying a chapter's section will provide you with the knowledge to complete the section's quiz and earn a badge.

In order to complete each quiz you will need to access and use a virtual machine (VM) running Kali Linux. The VM provides you with an environment to experiment, try new things and apply your newly gained skills to answers the quiz's questions.

Kali Linux VM

As mentioned above, you will be required to use a virtual machine running Kali Linux OS, a Linux distribution especially designed for penetration testing. Kali comes with a large number of preinstalled penetration-testing programs and tools which we will use through a number of chapters.

Initial Setup

In order to be able to run a virtual machine, you first need to download the "Kali.ova" file and install VirtualBox. You can download VirtualBox from this link.

  1. Once you have installed and launched VirtualBox, click on File > Import Appliance and browse to the location where the Kali.ova file is located.
  2. Select Kali.ova and click Next.
  3. On the "Appliance Import Settings", click Import
You are now ready to use Kali VM which is listed on the menu on the left. You can launch the VM by double clicking on it.

Using Kali VM

The virtual machine image you have imported is preconfigured and set up to allow to complete the quizzes of HackHub. Before launching the VM for the first time, it highly reccomended to take a snapshot of its initial state. Snapshots in VirtualBox allow you to save the current state of a VM, serving as a backup, which can be then used to restore that state reverting any changes. This is extremely useful as it allows to experiment in a sandbox environment where you can always go back to a previous state in case something goes wrong. To take a snapshot, click on the arrow button right next to the Machine Tools button on the upper right corner of VirtualBox's main screen and then click on snapshots. Then, click on the take button to take a snapshot. You can take multiple snapshots which will be structured hierarchically. Remember you always have the option to discard the instance of the Kali VM you are using and create a new one using the image file.

In order to gain access to the machine, you will need to login with the following credentials:

  • Username: "hackhub"
  • Password: "hackhub"

Using this user account, you will have access to all the required files and tools that will be needed to guide your learning experience.
