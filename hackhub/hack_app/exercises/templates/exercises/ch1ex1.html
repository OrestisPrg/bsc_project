Users

A UNIX-like system's user can either be a regular user, an administrative user or a service account.

  • An administrative root account, commonly known as a superuser, is automatically created and has administrative privileges for all services, access to all commands and files on the system.
  • Regular users have the necessary privileges to perform standard tasks on a Linux computer such as running word processors, databases, and Web browsers. They can store files in their own home directories. Since regular users do not normally have administrative privileges, they cannot accidentally delete critical operating system configuration files.
  • Services such as Apache, Squid, mail, games, and printing have their own individual service accounts. These accounts exist to allow each of these services to interact with your computer.

The Command-Line Interface (CLI)

As opposed to the widely known and used Graphical User Interface (GUI), the CLI is a very useful and powerful tool for users that seek simplicity, efficiency and speed. Although this might sound odd if you are not not familiar with CLI, this exercise will introduce you to fundamental concepts and usages of a Linux system's terminal.

The Command Prompt

By default, in a new terminal window, the user is at their home directory and can see a command prompt consisting of the following components [user]@[computer_name]:[current_path]$ where:

  • [user] is the username of the logged in user.
  • [computer_name] is the name of the computer user is logged in.
  • [current_path] is where the directory path the user currently is.
  • The $ symbol indicates the user is a regular user that doesn't have root priviledges. A root user is denoted by a # symbol.


You must always perform tasks working as a regular user unless the directory or files you are working on require root priviledges. Working as root on regular directories/files will not only change their permissions to root, but prevents you from accidentally deleting or altering critical operating system files.

Terminal Commands

The general structure of the terminal's commands constists of 3 components : command [-option(s)] [argument(s)]
The command name is typed first, followed by any optional options and arguments.

  • Options can be either a single character preceeded by a hyphen (-) or a string proceeded by 2 hyphens (--). Most commands provide a help option (usually -h or --help) that lists and demonstrates the usage of the available options and arguments.
    Note: some options require their own arguments. Their usage can be also demonstrated with the help option.
  • Arguments are usually used to specify the location of a file or directory by providing its path or any other associated information such as a file's or directory's name.

Basic Commands

  • ls lists the files and directories located in the current directory
    Options:
    • -a lists all entries including hidden entries (starting with a period) and the parent directory (..)
    • -l additionally displays the entries' permissions, links, owner, group, size, time and name.
